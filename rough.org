*Context switching* is the process where the kernel saves the state of the current
*process and loads the state of another process so the CPU can switch between them*.
The kernel is responsible for context switching.

Context Switching (Simple Steps)

1. *Process running in user mode*
   - A process is running normally in *user mode* on the CPU.
2. *Time slice finishes (Interrupt)*
   - The CPU timer notices that the process has used its allotted time.
   - The CPU interrupts the process and switches to *kernel mode*.
3. *Kernel saves the process state*
   - The kernel *saves the current state* of the process.
   - This includes CPU registers, program counter, and memory information.
   - This helps the process *resume later from the same point*.
4. *Kernel handles pending tasks*
   - The kernel checks and performs tasks that occurred during the time slice.
   - Example: *I/O operations* like keyboard input, disk read/write, etc.
5. *Kernel selects the next process*
   - The kernel checks the *ready queue (list of ready processes)*.
   - It chooses the *next process to run* using a scheduling algorithm.
6. *Prepare memory and CPU for new process*
   - The kernel *loads the memory context* of the new process.
   - It also prepares the *CPU registers* for that process.
7. *Set new time slice*
   - The kernel tells the CPU *how long this new process can run*.
8. *Switch back to user mode*
   - The CPU switches from *kernel mode to user mode*.
9. *New process starts running*
   - The selected process now *gets control of the CPU* and begins execution.

Memory Management (During Context Switch)
The *kernel also manages memory* when switching from one process to another.
Some important conditions must be followed.

Important Conditions
1. *Kernel Private Memory*
   - The kernel must have its *own private memory area*.
   - *User processes cannot access it*.
2. *Separate Memory for Each Process*
   - Every process gets *its own section of memory*.
3. *Process Isolation*
   - One process *cannot access the private memory of another process*.
4. *Memory Sharing*
   - Some memory can be *shared between processes* if needed.
5. *Read-Only Memory*
   - Some memory areas are *read-only*, meaning processes *can read but cannot modify* them.
6. *Using Disk as Extra Memory*
   + The system can use *disk space as additional memory* when RAM is not enough.
   + This is called *virtual memory*.


Virtual Memory and MMU
Modern CPUs have a component called *Memory Management Unit (MMU)*.

How It Works
1. A process *does not access physical memory directly*.
2. The kernel makes each process *feel like it has its own full memory*.
3. When the process tries to access memory:
   - The *MMU intercepts the request*.
4. The MMU *translates the virtual address* (process view) into a *physical address* (actual RAM location).
5. The kernel *creates and maintains the memory mapping*.



Context Switch and Memory
During a context switch:
1. The kernel stops the current process.
2. It changes the memory map from the old process.
3. It loads the memory map of the new process.
4. The new process then accesses its own memory space.


Page Table
- The structure used to store the memory address mapping is called a page table.

#+begin_src c
#include <stdio.h>
#include <string.h>

int main() {
  char str[] = "ABC";
  int n = strlen(str);
  int total = 1 << n;   // 2^n

  for (int mask = 0; mask < total; mask++) {
    for (int i = 0; i < n; i++) {
      if (mask & (1 << i))
	printf("%c", str[i]);
    }
    printf("\n");
  }
}
